当前可以想到的共MainAgent调用的SubAgent可以有：
1. 查找确定摄像头的智能体，这个应该就是连接了各种mcp可供查询摄像头的服务
   产出的是：1）给用户显示的确定的摄像头列表，包括离线和在线 2）传给下个智能体的是每个摄像头的详细信息，比如：
     - 摄像头的IP地址
     - 摄像头的端口号
     - 摄像头的用户名和密码
     - 摄像头的支持的视频编码格式
     - 摄像头的支持的视频分辨率
2. 视频数据预处理的智能体，这个应该是各种视频处理的skills，如：
   - 读取离线的视频数据，进行预处理，比如去噪、去模糊等
   - 对视频数据（在线或离线）进行抽帧，提取出关键帧
   - 对视频数据进行分类，比如人、车、动物等
   - 对视频数据进行目标检测，比如人、车、动物等
   - 对视频数据进行目标跟踪，比如人、车、动物等
   - 对视频数据进行行为识别，比如人在做什么动作等
   - 对视频数据进行事件识别，比如人在做什么事件等
   - 对视频数据的异常检测，比如异常的行为、异常的事件等
3. 针对预处理之后的系统，进行用户query各种特征提取后的查询
   - 针对用户的请求进行充分的分析，并
   - 用户可以查询视频数据中的各种特征，比如：
     - 视频数据中的人、车、动物等
     - 视频数据中的目标框、行为标签等
     - 视频数据中的异常行为、异常事件等
4. 视频数据后处理的智能体，这个应该是各种视频处理的skills，如：
   - 对视频数据进行可视化，比如绘制目标框、行为标签，轨迹绘制等
   - 对视频数据进行存储，比如保存到本地、上传到云存储等


主Agent可以：
1. 接收用户的query
2. 分析用户的query，判断需要调用哪些sub-agent
3. 调用对应的sub-agent，获取处理结果
4. 对sub-agent的处理结果进行整合，生成最终的结果
5. 将最终的结果返回给用户

加载各种图片和视频处理和分析的skills，进行基本的分析和处理，涉及到流程编排的时候，调用handoff的模式来进行子Agent的协调和管理
