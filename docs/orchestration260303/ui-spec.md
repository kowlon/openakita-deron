## 2. 区域详细说明

### 2.1 左侧边栏 (Left Sidebar)

**宽度**: `256px` (固定)
**背景色**: `#111722`
**功能**: 导航、会话管理、最佳实践入口

#### 2.1.1 结构层次 (从上到下)

| 序号 | 组件 | 说明 |
|------|------|------|
| 1 | **Logo 区域** | 图标 + "SeeAgent" 标题 + "AI Orchestrator" 副标题 |
| 2 | **New Chat 按钮** | 主色调背景，点击创建新会话 |
| 3 | **Best Practices 区域** | 可复用的任务模板入口 |
| 4 | **搜索框** | 搜索历史会话 |
| 5 | **会话列表** | 可滚动，显示活跃会话和历史会话 |
| 6 | **底部导航** | Settings + 用户头像/名称 |

#### 2.1.2 Best Practices 区域

```
┌─────────────────────────┐
│ BEST PRACTICES    [i]   │  ← 标题行，[i] 为信息图标
├───────────┬─────────────┤
│ [code]    │ [database]  │
│ Code      │ Extraction  │  ← 2x2 网格布局
│ Review    │             │
├───────────┼─────────────┤
│ [trending]│ [security]  │
│ Market    │ Security    │
└───────────┴─────────────┘
```

**组件属性**:
- 布局: `grid-cols-2` (2列网格)
- 每个按钮: 图标 + 文字标签
- 交互: 悬停时背景变为 `primary/20`，边框变为 `primary/50`

#### 2.1.3 会话列表

**活跃会话样式**:
- 背景: `bg-primary/20`
- 边框: `border-primary/30`
- 左侧图标背景: `bg-primary`

**历史会话样式**:
- 背景: 透明
- 悬停: `hover:bg-[#232f48]`
- 左侧图标背景: `bg-[#232f48]`

**会话卡片内容**:
```
┌──────────────────────────┐
│ [图标] Web Research Task │  ← 标题
│        12 steps • 2m ago │  ← 步骤数 + 时间
└──────────────────────────┘
```

### 2.2 中间主区域 (Main Chat Area)

**宽度**: `flex-1` (自适应)
**功能**: 对话交互、任务卡片展示

#### 2.2.1 消息布局

**用户消息** (右对齐):
```
                                    ┌──────────────────────────┐
                                    │ 用户消息内容 (蓝色气泡)    │
                                    └──────────────────────────┘
                                         You • 10:24 AM
```

**Agent 回复** (左对齐):
```
┌────┐
│ 🤖 │  ┌──────────────────────────┐
└────┘  │ Agent 回复内容            │
        └──────────────────────────┘
```

#### 2.2.2 任务卡片 (Task Card)

当用户触发最佳实践时，在对话流中嵌入任务卡片：

```
┌─────────────────────────────────────────────────────┐
│ █ ACTIVE BEST PRACTICE                              │
│ █ Web Research Task                   [⏸ PAUSED]   │
│ █                                                   │
│ █ ████████████░░░░░░░░░░░░░░░░░░  Step 2 of 4     │
│ █                                                   │
│ █ [头像]                    Open Details →         │
└─────────────────────────────────────────────────────┘
┌─────────────────────────────────────────────────────┐
│ PRELIMINARY INSIGHTS                                │
│ 初始发现摘要内容...                                  │
└─────────────────────────────────────────────────────┘
```

**卡片元素**:
| 元素 | 说明 |
|------|------|
| 左侧竖线 | `bg-primary`，宽度 1px，表示任务状态 |
| 状态标签 | PAUSED/RUNNING/COMPLETED，不同颜色 |
| 进度条 | 显示当前步骤进度 |
| 步骤计数 | "Step X of Y" 格式 |
| Preliminary Insights | 蓝色背景区域，显示初步结果 |

#### 2.2.3 输入栏 (Input Bar)

**位置**: 固定在主区域底部
**样式**: 渐变背景遮罩 + 输入框 + 发送按钮

### 2.3 右侧任务面板 (Right Task Panel)

**宽度**: `450px` (固定)
**功能**: 任务详情、步骤执行、结果编辑

#### 2.3.1 结构层次 (从上到下)

```
┌───────────────────────────────────────┐
│ [📊] 任务名称                    [✕]  │  ← Header
├───────────────────────────────────────┤
│ [▶] Cancel Task                       │  ← 操作按钮
├───────────────────────────────────────┤
│ ○ Step 1 → ● Step 2 → ○ Step 3 → ○   │  ← 横向步骤条
│   Done      Active      Pending       │
├───────────────────────────────────────┤
│                                       │
│   ACTIVE EXECUTION                    │  ← 执行状态区
│   [数据源卡片列表...]                  │
│                                       │
│   PRELIMINARY INSIGHTS                │  ← 初步洞察区
│   [摘要内容...]                        │
│                                       │
│   OUTPUT RESULTS              [编辑]  │  ← 结果输出区
│   [可编辑文本框]                       │
│                                       │
├───────────────────────────────────────┤
│ ← Previous Step    Next Step →       │  ← 底部导航
└───────────────────────────────────────┘
```

#### 2.3.2 横向步骤条 (Step Timeline)

**布局**: 水平排列，可横向滚动

**步骤状态样式**:

| 状态 | 图标背景 | 文字颜色 | 透明度 |
|------|----------|----------|--------|
| COMPLETED | `bg-emerald-500` (绿色) | `text-slate-500` | 60% |
| RUNNING | `bg-primary` (蓝色) + 环形高亮 | `text-primary` | 100% |
| PENDING | `bg-slate-300` (灰色) | `text-slate-500` | 40% |

**当前步骤标记**: 底部有一条蓝色指示线 (`h-1 bg-primary`)

#### 2.3.3 Active Execution 区域

显示当前步骤正在执行的操作：

```
┌─────────────────────────────────────┐
│ ACTIVE EXECUTION          2 Sources │
├─────────────────────────────────────┤
│ 🟢 mhealthintelligence.com    [▼]  │  ← 数据源卡片
│ 🟢 gartner.com                [▼]  │
└─────────────────────────────────────┘
```

- 绿色圆点: `animate-pulse` 动画，表示活跃
- 展开图标: 悬停时显示

#### 2.3.4 Output Results 区域

**功能**: 显示和编辑步骤输出

**元素**:
1. 标题行: "OUTPUT RESULTS" + 编辑图标 + 保存按钮
2. 可编辑文本框: `textarea`，支持直接编辑
3. 最小高度: `300px`

## 3. 区域间交互逻辑

### 3.1 左侧边栏交互

```
用户操作                          系统响应
─────────────────────────────────────────────────────
点击 Best Practice 按钮    →    创建新任务，中间区域显示任务卡片，
                                 右侧面板打开并显示任务详情

点击会话列表项              →    切换到该会话，加载历史消息，
                                 若有活跃任务则右侧面板同步显示

点击 New Chat              →    创建新会话，清空中间区域，
                                 关闭右侧面板（或显示空白状态）

搜索会话                   →    过滤会话列表，高亮匹配项
```

### 3.2 中间主区域交互

```
用户操作                          系统响应
─────────────────────────────────────────────────────
发送消息                   →    消息添加到对话流，
                                 Agent 判断是否命中任务路由

点击任务卡片 "Open Details" →   右侧面板滑入/激活，显示任务详情

用户输入与任务相关          →    路由到 SubAgent，更新步骤状态

用户输入与任务无关          →    走普通对话流程，计数器+1
                                 (连续5次无关则触发自动暂停)
```

### 3.3 右侧任务面板交互

```
用户操作                          系统响应
─────────────────────────────────────────────────────
点击步骤卡片                →    下区内容切换到该步骤详情，
                                 内容从右侧滑入动画

点击 "Cancel Task"          →    任务状态置为 CANCELLED，
                                 面板关闭，返回普通对话模式

点击 "Previous Step"        →    切换到上一步骤详情
                                 (仅展示，不改变执行状态)

点击 "Next Step"            →    切换到下一步骤详情
                                 (当前步骤完成后才可点击)

编辑 Output Results         →    文本框进入编辑模式，
                                 显示保存按钮，点击保存更改
```

### 3.4 状态联动关系

```
┌─────────────┐     创建任务      ┌─────────────┐
│  左侧边栏    │ ───────────────→ │  中间区域    │
│ Best Practice│                  │  任务卡片    │
└─────────────┘                   └──────┬──────┘
                                         │
                                         │ 同步
                                         ↓
                                  ┌─────────────┐
                                  │  右侧面板    │
                                  │  任务详情    │
                                  └─────────────┘
                                         │
           ┌─────────────────────────────┼─────────────────────────────┐
           ↓                             ↓                             ↓
    步骤状态更新                   任务状态变更                  输出结果编辑
           │                             │                             │
           ↓                             ↓                             ↓
    更新进度条/步骤条              更新状态标签                 更新任务卡片
    (中间区域同步)                (三区域同步)                (中间区域同步)
```

## 4. 状态显示规范

### 4.1 任务状态标签

| 状态 | 显示文字 | 背景色 | 文字色 |
|------|----------|--------|--------|
| RUNNING | RUNNING | `bg-primary/10` | `text-primary` |
| PAUSED | PAUSED | `bg-yellow-500/10` | `text-yellow-600` |
| COMPLETED | COMPLETED | `bg-emerald-500/10` | `text-emerald-600` |
| FAILED | FAILED | `bg-red-500/10` | `text-red-600` |
| CANCELLED | CANCELLED | `bg-slate-500/10` | `text-slate-500` |

### 4.2 步骤状态图标

| 状态 | 图标 | 颜色 |
|------|------|------|
| COMPLETED | `check` | `bg-emerald-500` |
| RUNNING | 步骤对应图标 | `bg-primary` + 环形光晕 |
| PENDING | 步骤对应图标 | `bg-slate-300` |
| FAILED | `error` | `bg-red-500` |
| SKIPPED | `skip_next` | `bg-slate-400` |

## 5. 响应式行为

|----------|----------|----------|----------|
| ≥ 1280px | 显示 (256px) | flex-1 | 显示 (450px) |

| < 768px | 默认隐藏，汉堡菜单 | 100% | 全屏弹层 |
