# 多任务编排需求整理

> 版本: v1.3  
> 日期: 2026-03-06  
> 状态: 需求确认（已合并迭代）

---

## 1. 背景与目标

- 建立一套多任务编排机制，支持 MainAgent 识别最佳实践场景并拆解为多步骤任务。
- 通过 SubAgent 完成步骤任务，并支持用户在过程中编辑、确认、切换与恢复任务焦点。
- 提供前端可视化流程与任务卡片交互，提升任务执行可控性与可理解性。
- 将“规则匹配路由”升级为“LLM 上下文路由”，并引入多会话任务激活管理。

## 2. 概念定义

### 2.1 核心角色

| 角色 | 定义 | 能力范围 |
|------|------|---------|
| **MainAgent** | 主代理，加载全量技能 | 全量工具、默认 prompt、消息路由、任务管理 |
| **SubAgent** | 步骤执行代理，独立 Agent 实例 | 受限工具、专用 prompt、独立对话历史 |
| **TaskSession** | 任务会话，管理任务生命周期 | 步骤调度、上下文传递、SubAgent 管理 |

### 2.2 MainAgent 与 SubAgent 路由关系（v1.3）

```
用户消息
  │
  ▼
MainAgent 编排入口
  │
  ├── LLM 判定: 显式激活任务 / 已有任务 / 新场景 / 普通对话
  │
  ├── 命中任务模式 → 路由到目标 TaskSession + 目标步骤 SubAgent
  │                     ├── 未激活步骤: 先激活
  │                     └── 输入不满足: AskUser
  │
  └── 普通对话 → MainAgent 自由对话链路
```

### 2.3 SubAgent 与 WorkerAgent 的关系

**统一架构，通过配置区分：**

| 维度 | SubAgent | WorkerAgent |
|------|----------|-------------|
| 进程 | 独立进程（步骤执行模式） | 独立进程 |
| Brain | 共享模型配置/代理 | 独立 |
| 工具集 | 受限 (step_def.tools，ToolExecutor 工具名) | 可配置 |
| Prompt | 专用 (step_def.system_prompt) | 可配置 |
| 生命周期 | 任务期间 | 长期运行 |
| 通信 | ZMQ | ZMQ |

### 2.4 Prompt 管理边界（v1.3）

- 独立管理两类编排 Prompt：
  - MainAgent -> SubAgent 路由判定 Prompt
  - 最佳实践场景触发判定 Prompt
- SubAgent 步骤 Prompt 继续保留在场景 YAML（`step_def.system_prompt`）。

## 3. 核心业务规则

### 3.1 场景触发与任务创建

- 用户通过对话触发时，由 LLM 判定是否命中最佳实践场景。
- 用户通过最佳实践入口触发时，直接创建任务并进入步骤执行流程。
- 新触发任务可抢占当前会话焦点，旧任务进入出域状态。

### 3.2 任务激活与焦点规则

- 系统同一时刻仅允许一个全局焦点任务（用于任务态路由）。
- 同一会话可存在多个任务，但仅一个会话激活任务。
- 激活优先级：
  1. 用户显式激活（卡片点击/明确指令）
  2. LLM 判定消息属于已有任务
  3. LLM 判定需新建最佳实践任务
  4. 普通对话

### 3.3 任务内步骤路由规则

- 活跃任务内由 LLM 判定目标步骤。
- 若目标步骤对应 SubAgent 未激活，系统先激活后执行。
- 若输入条件不满足，系统进入 `ASK_USER` 交互而非直接执行。
- 步骤输出在用户确认后写入上下文并传递到后续步骤。

## 4. 状态模型

### 4.1 生命周期状态

- PENDING
- RUNNING
- WAITING_USER
- COMPLETED
- CANCELLED
- FAILED

### 4.2 激活状态（新增）

- ACTIVE：当前会话焦点任务
- EXPLICIT_ACTIVE：用户显式激活任务
- OUT_OF_FOCUS：被新任务抢占后暂时出域
- INACTIVE：会话内存在但未激活

## 5. 子代理与步骤执行机制

### 5.1 SubAgent 核心特性

- 独立 Agent 实例，具备完整推理能力。
- tools/mcp_tools 为可执行集合，skills 仅影响提示词与技能调用能力。
- 使用步骤专用 prompt，按步骤职责执行。
- 与 MainAgent 共享模型配置/Brain 代理，但各自维护独立 ReasoningEngine 与对话历史。

### 5.2 步骤间上下文传递

- 上一步输出写入 TaskSession 上下文。
- 后续步骤自动注入可追溯上下文并继续执行。
- 上下文传递必须可追踪、可回放。

### 5.3 超出能力范围处理

当用户请求超出当前 SubAgent 能力范围时：

1. SubAgent 识别能力边界并提示用户
2. 提供可选路径：
   - 借用其他工具（需确认）
   - 切换到其他步骤
   - 回到 MainAgent 自由模式

## 6. 用户交互与前端需求

### 6.1 会话与任务卡片展示

- 任务相关对话必须展示任务卡片。
- 同一任务在会话内首次展示大卡片（完整信息）。
- 同一任务后续轮次展示简洁卡片（轻量状态）。
- 普通聊天不展示任务卡片。

### 6.2 卡片交互能力

- 点击卡片激活对应任务。
- 卡片支持确认步骤、切换步骤、取消任务。
- 被抢占任务需有可见提示，并可被再次显式激活。

### 6.3 页面布局要求

- 左侧保留会话列表并展示最佳实践任务入口。
- 中间会话区支持任务卡片与对话混排。
- 右侧展示任务步骤状态与输出编辑区。

## 7. 后端能力需求

### 7.1 决策能力

- 路由主路径由 LLM 上下文决策驱动，替代规则关键词主路径。
- 决策至少覆盖：
  - 场景触发判定
  - 任务选择判定
  - 步骤选择判定
  - 输入满足性判定（`ASK_USER` 兜底）

### 7.2 激活管理能力

- 提供会话激活任务查询能力。
- 提供任务抢占、出域、显式激活、恢复能力。
- 提供跨会话隔离与一致性校验能力。

### 7.3 事件通知能力

后端输出结构化事件，驱动前端状态同步：

- `task_card_created`
- `task_card_compact`
- `task_activation_changed`
- `task_out_of_focus`

### 7.4 接口能力约束

- 任务激活接口：`POST /api/tasks/{task_id}/activate`
- 会话激活任务查询接口：`GET /api/sessions/{session_id}/active-task`
- 任务列表查询支持 `session_id` 与 `include_inactive` 参数

## 8. 非功能需求

### 8.1 可维护性

- 路由与场景触发 Prompt 独立维护，支持版本化管理。
- 决策逻辑与执行逻辑解耦，保持高内聚低耦合。
- SubAgent 步骤 Prompt 保持配置文件维护，避免改造面扩散。

### 8.2 可观测性

- 记录决策置信度、目标任务、目标步骤、决策理由。
- 记录任务激活切换链路，支持回溯与审计。

### 8.3 兼容性

- 灰度期间保留规则匹配降级开关。
- 保持现有任务创建、确认、取消流程可用。

## 9. 验收标准

### 9.1 业务验收

- 有活跃任务时触发新最佳实践，系统可正确抢占并将旧任务置为 OUT_OF_FOCUS。
- 用户点击历史任务卡片后，可立即成为激活任务并接管后续任务对话。
- 同任务在聊天区实现“首大后简”的卡片展示策略。

### 9.2 技术验收

- 路由主路径不依赖规则关键词匹配。
- 场景触发与步骤路由均有 LLM 决策记录。
- 前后端对任务激活状态一致。

## 10. 迁移与上线要求

### 10.1 灰度策略

1. 先启用任务激活管理器（不切主路由）
2. 再启用场景触发 LLM 判定
3. 最后启用步骤路由 LLM 判定并关闭规则主路径

### 10.2 回滚策略

- 任一阶段可切回规则路由
- 激活管理可回退为单映射会话任务模型

## 11. 与现有架构关系

### 11.1 HandoffAgent 范围

- 不使用 HandoffAgent 作为本方案主路径。
- 保持 MainAgent + TaskSession + SubAgent 的编排路径。

### 11.2 组件复用

| 组件 | 来源 | 复用方式 |
|------|------|---------|
| Brain | MainAgent | 共享模型配置/代理 |
| ReasoningEngine | 新建 | 每个 SubAgent 独立创建 |
| ToolExecutor | WorkerAgent/Agent | 与 MainAgent 一致执行器 |
| TaskState | AgentState | 扩展使用 |

## 12. 术语补充

- **任务激活**：任务被选为当前会话路由目标的状态
- **出域**：任务仍存在但不再是当前焦点任务
- **显式激活**：由用户主动操作触发的激活，不依赖自动判定
- **任务相关对话**：被判定为针对任务/步骤的用户消息
