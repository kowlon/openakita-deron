# 多任务编排需求整理

> 版本: v1.1
> 日期: 2026-03-03
> 状态: 需求确认

---

## 1. 背景与目标
- 建立一套多任务编排机制，支持主代理识别"最佳实践场景"并拆解为多步骤任务。
- 通过子代理完成步骤任务，并支持用户在过程中编辑与确认输出。
- 提供前端可视化流程与交互，便于用户理解与控制任务进度。

## 2. 概念定义

### 2.1 核心角色

| 角色 | 定义 | 能力范围 |
|------|------|---------|
| **MainAgent** | 主代理，加载全量技能 | 全量工具、默认 prompt、消息路由、任务管理 |
| **SubAgent** | 步骤执行代理，独立 Agent 实例 | 受限工具、专用 prompt、独立对话历史 |
| **TaskSession** | 任务会话，管理任务生命周期 | 步骤调度、上下文传递、SubAgent 管理 |

### 2.2 SubAgent 与 MainAgent 的关系

```
MainAgent (消息路由器)
    │
    ├── 无活跃任务 → 普通对话 / 场景匹配
    │
    └── 有活跃任务 → 路由到当前 SubAgent
                         │
                         ├── chat_with_step(step_id, message)
                         └── 共享 Brain，独立 ReasoningEngine
```

### 2.3 SubAgent 与 WorkerAgent 的关系

**统一架构，通过配置区分：**

| 维度 | SubAgent | WorkerAgent |
|------|----------|-------------|
| 进程 | 同进程 | 独立进程 |
| Brain | 共享 | 独立 |
| 工具集 | 受限 (step_def.tools) | 可配置 |
| Prompt | 专用 (step_def.system_prompt) | 可配置 |
| 生命周期 | 任务期间 | 长期运行 |
| 通信 | 直接调用 | ZMQ |

## 3. 核心业务流程

### 3.1 用户通过对话触发
- 用户提出任务后，MainAgent 判断是否命中最佳实践场景。
- 命中后进入任务创建流程，并在必要环节询问用户确认。
- 创建任务时，将用户聊天内容作为初始上下文传递给第一个 SubAgent。

### 3.2 用户通过最佳实践入口触发
- 用户直接点击最佳实践入口创建任务。
- 不需要对话匹配，直接进入任务执行。
- 所有 SubAgent 的输入需要用户交互输入。

### 3.3 MainAgent 消息路由机制

```
用户消息
    │
    ▼
MainAgent.chat(message)
    │
    ├── 检查是否有活跃任务
    │       │
    │       ├── 有任务 + step 模式 → 路由到 SubAgent
    │       │       └── TaskSession.chat_with_step(current_step_id, message)
    │       │
    │       └── 无任务 → 尝试场景匹配
    │               │
    │               ├── 匹配成功 → 创建任务，进入 SubAgent
    │               │
    │               └── 不匹配 → 普通对话
    │
    └── 返回响应给用户
```

### 3.4 前端显示

前端显示当前正在与哪个 SubAgent 对话：
- 显示 "正在与 [步骤名称] SubAgent 对话"
- 显示该 SubAgent 的能力描述
- 显示步骤进度 (2/4)

## 4. 子代理与步骤执行机制

### 4.1 SubAgent 核心特性

- **独立 Agent 实例**: 不是配置对象，而是真正的 Agent
- **受限工具集**: 只能使用 step_def.tools 中定义的工具
- **专用 prompt**: 使用 step_def.system_prompt
- **共享 Brain**: 与 MainAgent 共享 LLM 客户端，节省资源
- **独立 ReasoningEngine**: 拥有专用的推理引擎
- **独立对话历史**: 每个 SubAgent 维护自己的 messages

### 4.2 步骤间上下文传递

**自动注入机制：**

```
Step 1 (analyze) 完成
    │
    │ output = {analysis: "代码分析结果..."}
    │
    ▼
TaskSession.context["analysis"] = output
    │
    ▼
Step 2 (review) 开始
    │
    │ SubAgent 系统提示词:
    │ "你是代码审查专家...
    │
    │  ## 前置步骤输出
    │  ### analysis
    │  代码分析结果..."
    │
    ▼
SubAgent 基于上下文继续工作
```

### 4.3 超出能力范围的处理

当用户请求超出当前 SubAgent 能力范围时：

1. SubAgent 识别请求超出工具范围
2. 告知用户当前步骤的能力边界
3. 提供选项：
   - 借用其他工具（需要用户确认）
   - 切换到其他步骤
   - 切换到自由模式（使用 MainAgent 全量能力）

## 5. 用户交互与编辑机制

### 5.1 编辑机制
- 支持 Edit 模式：步骤输出可由用户编辑并确认。
- 用户可通过对话对步骤输出进行调整。
- 用户点击"确认"或"下一步"后，保存编辑并作为下一步输入。

### 5.2 步骤切换
- 用户可显式切换到任意步骤
- 切换时检查步骤执行条件
- 不满足条件时提示用户

### 5.3 任务取消
- 支持"取消任务"按钮
- 取消时销毁所有 SubAgent
- 取消时清理任务状态

## 6. 前端需求整理

整体页面与现有 webapp 的 seeagent 布局一致，分成三部分。

### 6.1 最左侧（上下两部分）

#### 上半部分
- 对话列表上方新增"最佳实践任务列表"，最多显示 5 行。
- 提供"更多"入口跳转到右侧区域展示全部最佳实践卡片。

#### 下半部分
- 现有的会话列表。

### 6.2 中间部分（会话区）
保持现有对话方式，新增一种任务卡片类型。
- 当在主对话框（MainAgent）聊天触发某场景的最佳实践时，显示任务卡片，包含简短描述与SubAgent，确认后右侧显示任务部分。
- 直接点击最佳实践时，启动一个主对话框界面（MainAgent）并生成任务卡片，右侧直接显示任务部分，同时引导用户开始输入第一步任务依赖。

### 6.3 右侧部分
- 正常步骤卡片：显示步骤卡片的输出内容。
- 任务卡片：分为上下两部分。
  - 上面部分：该任务卡片包含的多个 SubAgent 名字、基本说明与状态。
  - 下面部分：仅展示最后一步的输出结果，支持编辑并保存。
  - 步骤总结不在右侧展示，统一在主对话框显示，并标注该总结对应的步骤输出（带步骤标识说明）。

## 7. 后端逻辑
1) 主对话框里的聊天（MainAgent）触发最佳实践
- 动态创建最佳实践对应的多个 SubAgent，每个 SubAgent 负责一个步骤。
- 每个 SubAgent 的输入需要用户交互输入，用户根据说明输入必要依赖信息。
- 用户点击确认后启动第一个 SubAgent，开始处理第一个步骤。
- 第一个步骤输出内容在用户确认进入下一步后，作为上下文传递给下一个 SubAgent。
2) 在左侧直接点击最佳实践
- 相当于打开一个对话框，创建 MainAgent。
- 其他流程同上。

## 8. 状态与约束

### 7.1 任务状态
- PENDING: 等待开始
- RUNNING: 执行中
- WAITING_USER: 等待用户确认
- COMPLETED: 已完成
- CANCELLED: 已取消

### 7.2 生命周期约束
- 任务完成后，所有 SubAgent 被销毁
- 任务取消后，清理所有状态
- 上一步输出必须可追溯并作为下一步输入

### 7.3 资源约束
- SubAgent 与 MainAgent 共享 Brain
- 每个 SubAgent 创建独立的 ReasoningEngine
- 受限工具通过 RestrictedToolExecutor 实现

## 8. 与现有架构的关系

### 8.1 不再使用 HandoffAgent

| 维度 | HandoffAgent | 本设计的 SubAgent |
|------|-------------|-------------------|
| 切换机制 | LLM tool_call 触发 | 用户显式切换 |
| 执行引擎 | brain.messages_create | ReasoningEngine |
| 消息历史 | 共享 | 独立 |
| 与设计关系 | **不使用** | 新设计 |

### 8.2 复用现有组件

| 组件 | 来源 | 复用方式 |
|------|------|---------|
| Brain | MainAgent | 共享引用 |
| ReasoningEngine | 新建 | 每个 SubAgent 独立创建 |
| ToolExecutor | MainAgent | 包装为 RestrictedToolExecutor |
| TaskState | AgentState | 扩展使用 |
