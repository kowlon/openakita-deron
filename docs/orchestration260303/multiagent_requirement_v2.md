# MainAgent 任务模式需求文档

## 1. 背景与目标

### 1.1 现状
当前 MainAgent 已具备两种能力：
- **普通 Agent Loop 对话**：标准的多轮对话交互
- **Plan 模式**：规划与方案设计

### 1.2 目标
新增第三种能力——**任务模式**，让 MainAgent 支持：
- 可编排的多步骤任务执行
- 任务状态的持久化与恢复
- 保留现有 ask_user 交互能力

---

## 2. 核心概念

### 2.1 最佳实践（BestPractice）
最佳实践是**可复用的任务模板配置**，定义了：
- 任务的元数据（名称、描述、适用场景）
- 触发条件描述（供 LLM 判定是否命中）
- 步骤序列定义

**使用场景**：用户触发最佳实践时，系统加载配置并实例化为运行时任务。

### 2.2 任务（Task）
任务是用户的**高级意图实例**，具有以下特性：
- 由多个步骤按顺序组成
- 具有唯一 ID，支持路由与恢复
- 关联最佳实践模板 ID（若从模板创建）

### 2.3 步骤（TaskStep）
步骤是任务的**执行单元**，对应一次 SubAgent 调用：
- 拥有独立的输入/输出
- 支持状态追踪（PENDING/RUNNING/COMPLETED/FAILED）
- 可产生制品（文件、代码片段等）

### 2.4 子代理（SubAgent）
SubAgent 是执行单步骤逻辑的**专用 Agent 实例**：
- 通过配置定义身份、能力和运行参数
- 默认加载系统工具，按需加载 MCP 与 Skills
- 支持通过 ask_user 补齐输入

### 2.5 会话任务集（SessionTasks）
管理单个会话内所有任务的状态：
- 维护任务集合与活跃任务
- 提供路由索引与上下文管理
- 确保同一时刻仅一个活跃任务

---

## 3. 功能需求

### 3.1 任务触发

| 入口 | 触发方式 | 流程 |
|------|----------|------|
| 最佳实践入口 | 用户直接点击 | 直接创建任务并进入步骤执行 |
| 对话触发 | LLM 上下文判定 | 命中则创建任务；不命中则走普通对话 |
| Plan 转任务 | 用户明确要求 | 将 Plan 步骤映射为 Task 步骤 |

### 3.2 输入路由

MainAgent 根据用户输入判断路由目标：
- **命中活跃任务步骤** → 路由到对应 SubAgent
- **未命中** → 走普通 Agent Loop 对话流程

### 3.3 活跃任务管理

**单活跃任务原则**：
- 同一时刻仅允许一个活跃任务
- 会话中可存在多个任务，依据上下文激活

**自动暂停机制（Auto-Suspend）**：
- 连续 5 次对话被判定为与任务无关时，自动暂停任务
- 任务状态置为 PAUSED，销毁内存中的任务实例
- 用户需显式点击"恢复"才能继续

### 3.4 任务生命周期

```
创建 → 执行 → [暂停 → 恢复] → 完成/失败/取消
```

支持：
- 跨会话恢复（基于持久化存储）
- 步骤级状态追踪
- 用户反馈与步骤回退

---

## 4. 交互需求

### 4.1 最佳实践入口
- 会话列表上方展示最佳实践入口
- 默认显示 5 个，支持展开查看更多

### 4.2 任务看板

**布局**：
- 上区：横向展示所有步骤（步骤名 + 描述）
- 下区：当前步骤的详细输出内容

**交互**：
- 点击步骤卡片，下区内容从右侧滑入
- 支持步骤前进/回退（点击或对话触发）
- 暂停状态显示"任务已暂停，点击恢复"提示

### 4.3 步骤输出编辑
- 文本输出：支持直接编辑
- 数组输出：元素卡片化展示，支持删除/编辑

### 4.4 任务恢复
- 点击任务卡片或"恢复"按钮
- 系统重新加载任务结构与历史数据
- 激活任务状态，继续执行

---

## 5. 约束与原则

### 5.1 功能约束
| 约束项 | 说明 |
|--------|------|
| 单活跃任务 | 同一时刻仅一个任务处于活跃状态 |
| 顺序执行 | 步骤按定义顺序依次执行 |
| 状态持久化 | 任务状态实时保存，支持跨会话恢复 |

### 5.2 交互原则
- MainAgent 负责路由、状态管理与展示协调
- SubAgent 专注单步骤逻辑，通过 ask_user 补齐输入
- 用户反馈实时反映到步骤输出

---

## 6. 状态定义

### 6.1 任务状态
| 状态 | 说明 |
|------|------|
| PENDING | 已创建，等待执行 |
| RUNNING | 正在执行中 |
| PAUSED | 已暂停（自动或手动） |
| COMPLETED | 已完成 |
| FAILED | 执行失败 |
| CANCELLED | 已取消 |

### 6.2 步骤状态
| 状态 | 说明 |
|------|------|
| PENDING | 等待执行 |
| RUNNING | 正在执行 |
| COMPLETED | 已完成 |
| FAILED | 执行失败 |
| SKIPPED | 已跳过 |

---

## 7. 非功能性需求

### 7.1 持久化
- 任务状态实时持久化到 SQLite
- 支持历史会话的任务恢复
- 存储层级覆盖 SessionTasks、Task、Step 三级

### 7.2 性能
- 单机部署，MainAgent 与 SubAgents 同进程运行
- 优先使用进程内通信，避免网络开销
- 大数据采用引用传递，延迟加载
