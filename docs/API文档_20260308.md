## WebApp 重点接口速查

> **说明**: 以下接口是 `seeagent-webui` 前端应用的核心依赖接口，建议优先熟悉。

| 接口 | 方法 | 用途 | 使用场景 |
|------|------|------|----------|
| `/api/chat` | POST | **核心聊天** | 主要对话接口，SSE 流式返回 |
| `/api/chat/cancel` | POST | 取消任务 | 用户中断当前任务 |
| `/api/chat/skip` | POST | 跳过步骤 | 跳过当前工具执行 |
| `/api/chat/insert` | POST | 插入消息 | 运行中插入用户消息 |
| `/api/tasks` | GET/POST | **任务列表/创建** | 任务编排管理 |
| `/api/tasks/{id}` | GET | 任务详情 | 获取单个任务状态 |
| `/api/tasks/{id}/resume` | POST | 恢复任务 | 恢复暂停的任务 |
| `/api/tasks/{id}/pause` | POST | 暂停任务 | 暂停运行中的任务 |
| `/api/tasks/stats` | GET | 任务统计 | 仪表盘统计展示 |
| `/api/best-practices` | GET | **模板列表** | 最佳实践模板选择 |
| `/api/models` | GET | 模型列表 | 显示可用端点状态 |
| `/api/health` | GET | 健康检查 | 服务状态检测 |
| `/api/upload` | POST | 文件上传 | 上传图片/文档附件 |
| `/api/files` | GET | 文件服务 | 获取工作区文件 |
| `/api/stats/tokens/context` | GET | 上下文统计 | 显示当前 token 使用量 |

## 聊天接口

> **WebApp 核心接口**: 聊天相关接口是前端应用的主要交互入口

### POST /api/chat `WebApp核心`

主聊天端点，使用 SSE（Server-Sent Events）流式返回响应。

**请求体**:
```json
{
  "message": "请帮我分析这段代码",
  "conversation_id": "chat_abc123",
  "plan_mode": false,
  "endpoint": null,
  "thinking_mode": "auto",
  "thinking_depth": "medium",
  "attachments": [
    {
      "type": "image",
      "name": "screenshot.png",
      "url": "/api/uploads/123_screenshot.png",
      "mime_type": "image/png"
    }
  ]
}
```

**参数说明**:

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| message | string | 否 | 用户消息文本 |
| conversation_id | string | 否 | 会话ID，用于上下文关联 |
| plan_mode | boolean | 否 | 强制使用计划模式，默认 false |
| endpoint | string | 否 | 指定端点名称，null 表示自动选择 |
| thinking_mode | string | 否 | 思考模式：auto/on/off |
| thinking_depth | string | 否 | 思考深度：low/medium/high |
| attachments | array | 否 | 附件列表 |

**请求示例**:
```bash
curl -X POST http://127.0.0.1:18900/api/chat \
  -H "Content-Type: application/json" \
  -d '{"message": "你好", "conversation_id": "test_001"}'
```

**SSE 事件类型**:

| 事件类型 | 说明 |
|----------|------|
| thinking_start | 思考开始 |
| thinking_delta | 思考增量内容 |
| thinking_end | 思考结束 |
| text_delta | 文本增量输出 |
| tool_call_start | 工具调用开始 |
| tool_call_end | 工具调用结束 |
| plan_created | 计划创建 |
| plan_step_updated | 计划步骤更新 |
| ask_user | 向用户提问 |
| agent_switch | Agent 切换 |
| artifact | 生成产物通知 |
| error | 错误信息 |
| done | 完成（包含 usage 统计） |

**SSE 响应示例**:
```
data: {"type": "text_delta", "content": "你好"}

data: {"type": "done", "usage": {"input_tokens": 100, "output_tokens": 50, "total_tokens": 150}}
```

### POST /api/chat/insert

向运行中的任务上下文插入用户消息。智能路由：如果是停止/跳过命令，自动转发到 cancel/skip。

**请求体**:
```json
{
  "conversation_id": "chat_abc123",
  "message": "请暂停一下",
  "reason": ""
}
```

**请求示例**:
```bash
curl -X POST http://127.0.0.1:18900/api/chat/insert \
  -H "Content-Type: application/json" \
  -d '{"message": "停止", "conversation_id": "test_001"}'
```

**响应示例（识别为停止命令）**:
```json
{
  "status": "ok",
  "action": "cancel",
  "reason": "用户发送停止指令: 停止"
}
```

## 附录

### 错误响应格式

所有接口在出错时返回统一格式：

```json
{
  "error": "错误描述",
  "detail": "详细错误信息"
}
```

### HTTP 状态码

| 状态码 | 说明 |
|--------|------|
| 200 | 成功 |
| 201 | 创建成功 |
| 400 | 请求参数错误 |
| 403 | 权限不足 |
| 404 | 资源不存在 |
| 413 | 请求体过大 |
| 500 | 服务器内部错误 |
| 503 | 服务不可用（如组件未初始化） |

### CORS 配置

API 服务已启用 CORS，允许任意来源访问：
- `allow_origins: ["*"]`
- `allow_credentials: true`
- `allow_methods: ["*"]`
- `allow_headers: ["*"]`
