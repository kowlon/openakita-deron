# Plan模式UI展示方案

## 核心理念

### Plan模式 vs ReAct模式的区别

#### ReAct模式（现有）
```
用户请求 → LLM思考 → 决定工具1 → 执行 → 观察结果 →
         → LLM思考 → 决定工具2 → 执行 → 观察结果 →
         → LLM思考 → 决定工具3 → 执行 → 完成
```
**特点**：每一步都是实时决定，不知道下一步是什么

#### Plan模式（应该的样子）
```
用户请求 → LLM规划 → 生成完整计划（步骤1、2、3）→
         → 执行步骤1 → 更新状态 →
         → 执行步骤2 → 更新状态 →
         → 执行步骤3 → 更新状态 → 完成
```
**特点**：一开始就知道所有步骤，逐步执行并更新状态

## 推荐方案

### 最终推荐：方案C（Plan卡片 + 折叠步骤）

#### 理由
1. **信息层次清晰**
   - Plan卡片：整体视图，快速了解进度
   - 折叠步骤：详细信息，按需查看

2. **用户体验好**
   - 默认简洁，不会信息过载
   - 可以展开查看详情
   - 进度一目了然

3. **实现成本适中**
   - 不需要重复显示步骤卡片
   - Plan卡片内部管理所有步骤状态
   - 可以复用现有的步骤卡片组件（在右侧面板）

4. **符合Plan模式特点**
   - 一开始就显示完整计划
   - 实时更新每个步骤的状态
   - 清晰的进度指示

## 实现细节

### 数据结构

```typescript
interface PlanState {
  id: string
  task_summary: string
  status: 'in_progress' | 'completed' | 'failed'
  steps: PlanStep[]
  created_at: string
  completed_at?: string
}

interface PlanStep {
  id: string
  description: string
  tool?: string
  skills?: string[]
  status: 'pending' | 'in_progress' | 'completed' | 'failed' | 'skipped'
  result?: string
  started_at?: string
  completed_at?: string
  duration?: number
}
```

### 事件处理

```typescript
// 1. 接收 plan_created 事件
case 'plan_created': {
  const planData = eventRecord.plan as PlanState
  setActivePlan(planData)
  // 创建Plan卡片
}

// 2. 接收 plan_step_updated 事件
case 'plan_step_updated': {
  const stepId = eventRecord.step_id as string
  const status = eventRecord.status as string
  const result = eventRecord.result as string

  setActivePlan(prev => {
    if (!prev) return null
    return {
      ...prev,
      steps: prev.steps.map(s =>
        s.id === stepId
          ? { ...s, status, result, completed_at: Date.now() }
          : s
      )
    }
  })
}

// 3. 接收 tool_call_start/end 事件
// 可选：在右侧面板显示详细的工具执行信息
```

### 组件结构

```
PlanCard
├─ PlanHeader (标题、进度条)
├─ PlanStepList (步骤列表)
│  └─ PlanStepItem (单个步骤)
│     ├─ 状态图标
│     ├─ 步骤描述
│     ├─ 执行结果
│     └─ 查看详情按钮
└─ PlanFooter (展开/收起按钮)
```

## 对比总结

| 方案 | 优点 | 缺点 | 推荐度 |
|------|------|------|--------|
| A: Plan卡片 + 步骤卡片 | 信息完整 | 信息重复，占用空间大 | ⭐⭐⭐ |
| B: 仅Plan卡片 | 简洁集中 | 缺少详细的工具执行信息 | ⭐⭐⭐⭐ |
| C: Plan卡片 + 折叠步骤 | 平衡信息密度和可读性 | 实现稍复杂 | ⭐⭐⭐⭐⭐ |

**最终推荐：方案C**
