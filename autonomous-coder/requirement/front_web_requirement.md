# 前端页面需求概述

本文档描述 SeeAgent Web UI 前端页面的功能需求和实现规范，强调会话可视化与步骤可控。

## 一、会话可视化与步骤可控

### 1. 核心目标
- 每次会话中的每一步都必须可视化展示
- 每一步都必须可控，可在执行后决定是否进入下一步

### 2. 执行模式
- **Auto 模式**：每一步执行完成后自动进入下一步
- **Edit 模式**：每一步执行完成后进入可编辑状态，用户确认后进入下一步

### 3. 可编辑结果与卡片化展示
- 测试过程中可模拟某个 skill 的上一步输出为 JSON 数组
- JSON 数组的每个 item 以卡片形式展示
- 用户可删除任意卡片后点击确认进入下一步

### 2. 左侧会话列表
- 展示每次会话的主题摘要
- 显示消息数量和最后更新时间
- 支持搜索会话
- 点击某个会话后加载该会话的聊天记录
- 点击"+"按钮创建新会话
- 支持删除会话

### 3. 中间主区域

#### 3.1 内容展示

**中间区域只显示两类内容**：

1. **用户消息** - 显示用户提出的问题（右侧对齐，蓝色主题）
2. **步骤卡片** - AI 回答以步骤卡片形式展示（左侧对齐）

**不再显示传统的助手消息**，所有回答都通过步骤卡片呈现。

#### 3.2 步骤卡片展示

**每个步骤卡片包含**：
- 状态图标（○ 等待中、⟳ 执行中、✓ 已完成、✕ 失败）
- 步骤类型图标（🧠 LLM调用、🔧 工具调用、⚡ 技能调用、💭 思考、📋 规划）
- 步骤标题
- 简要描述
- 右侧向下箭头按钮

**步骤状态表现**：
- 执行中：显示进度条和阶段说明
- 已完成：显示结果摘要
- 失败：显示错误信息

#### 3.3 步骤卡片两种交互

```
┌─────────────────────────────────────────┐
│  ✓ 🧠 步骤: 分析用户需求          ▼    │  ← 点击整张卡片 → 右侧显示详情
│  简要: 正在分析天气查询请求...          │
└─────────────────────────────────────────┘
     ↓ 点击右侧箭头展开
┌─────────────────────────────────────────┐
│  ✓ 🧠 步骤: 分析用户需求          ▲    │
│  简要: 正在分析天气查询请求...          │
│  ┌─────────────────────────────────┐   │
│  │ 执行结果概要（摘要区域）          │   │
│  │ - 识别意图: 天气查询             │   │
│  │ - 提取地点: 北京                 │   │
│  │ - 准备调用天气API...             │   │
│  └─────────────────────────────────┘   │
└─────────────────────────────────────────┘
```

| 点击位置 | 效果 |
|---------|------|
| 点击整张步骤卡片 | 右侧区域显示该步骤的**完整执行详情** |
| 点击右侧向下/向上箭头 | 卡片内部向下展开/收起，显示该步骤执行结果的**概览**（手风琴效果） |

#### 3.4 流式输出
- 步骤以流式输出方式逐步补齐
- 步骤数据内部携带步骤 ID，用于查询与绑定，但不在界面展示
- 同一步骤可多次更新进度与摘要内容
- Edit 模式下，步骤完成后进入可编辑态，显示确认入口

#### 3.5 输入框
- 支持多行输入（自动调整高度）
- 支持 Enter 发送，Shift+Enter 换行
- 发送按钮在输入时可用
- 响应流式传输时禁用输入

### 4. 右侧步骤详情面板

**触发方式**：点击中间区域的步骤卡片（整张）

**显示内容**：
- 步骤状态标签
- 步骤类型标签
- 标题和描述
- 开始/结束时间
- 执行时长
- 输入数据（JSON 格式）- 可选展示
- 输出结果（Markdown 渲染）
- 输出数据（JSON 格式）
- 错误信息（失败时）
- 复制结果内容按钮
- 关闭面板按钮

## 四、用户体验增强

### 1. 左侧会话摘要自动生成
- 当用户发送query并收到AI响应后，**自动生成会话主题摘要**
- 摘要应简洁概括用户的query意图（建议10-20字）
- 生成时机：第一个步骤完成或收到AI响应后
- 示例：用户发送"帮我查查曾德龙是谁，并帮我写入到pdf文件里"，摘要应为"查询曾德龙信息并输出PDF"

### 2. 原始用户query的正确展示
- 中间对话区域必须**准确显示用户发送的原始query**
- 不得将用户的中文query替换为英文或其他翻译版本
- 用户消息应实时更新：发送消息后立即在界面上显示

### 3. 核心步骤的聚合与过滤显示

**需求说明**：步骤展示应进行智能聚合，只显示核心业务步骤，而非所有底层操作。

**示例场景**：

用户发送query：`"帮我查查曾德龙是谁，并帮我写入到pdf文件里"`

**期望显示的步骤（4个核心步骤）**：
```
步骤1: 🧠 意图分析
       识别到需要搜索"曾德龙"相关信息并输出为PDF

步骤2: 🔍 搜索
       正在搜索"曾德龙"相关信息...

步骤3: 📄 PDF输出
       正在生成PDF文件...

步骤4: ✅ 总结
       已完成搜索并输出PDF文件，保存路径：/xxx/曾德龙.pdf
```

**不应显示的底层操作**：
- 读写临时文件
- 执行系统命令
- 加载配置文件
- 其他内部工具调用

**聚合原则**：将多个原子操作合并为一个有意义的业务步骤，让用户看到的是"做了什么"，而不是"怎么做的"。

## 三、长时运行步骤与进度展示

### 1. 触发方式
- 点击步骤卡片或展开摘要时，若该步骤为长时运行的 skill 或 MCP，则展示进度

### 2. 进度展示要求
- 显示阶段名称、当前阶段说明与进度条
- 支持持续更新进度与阶段说明
- 失败时展示错误信息与最后停留阶段

### 3. 依赖的技能与 MCP 改造要求
- skill 和 MCP 的函数需提供可持续更新的进度信息
- 进度信息必须与步骤 ID 绑定，便于前端更新同一步骤
